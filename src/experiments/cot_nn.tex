In this subsection the neural network training process with the processed COT dataset is described and the performance evaluated.

Even though the neural network presented in~\ref{subsec:nnmodel} is designed to be data independent, it was developed and tested mostly with the COT data.
On top of the presented architecture, some model tuning with the COT data in mind was done.

\subsubsection{Tuning the Neural Network With COT Data}
The training process was tailored to the characteristics and challenges related to the COT data while keeping in mind the underlying physical interpretation of advection fields.
The following choices were guided by performance of the model against the test dataset.

The preciously described architecture was chosen through extensive experimentations guided by performance, interprability, and model simplicity.
Within the ResNet model increasing the network depth provided little benefits and thus the network was kept quite simple.
The advection fields were produced using varioud basis types including Fourier, wavelet, and polynomial basis with different numbers of basis vectors.
The described second degree polynomials provided the best results in terms of advection field properties.
Furthermore, the image coordinates were placed within the positive real axes from 0 to 1 for evaluation of the polynomial functions.

Various loss functions were examined, including the MSE loss along with physics-based parameters accounting to the advection field properties such as gradient, curl, and divergence inspired by~\cite{debezenac}.
Also structural similarity (SSIM), multi-scale SSIM (MS-SSIM), logarithic cosh loss functions were tested.
Due to the small effect of the loss function in the results and no required parameters, the MSE loss was used for training the final model.

Three latest images proved to be the optimal amount of information for the network to produce advection fields.
The average loss was then computed over four future time steps.
Given the noisy nature of the data, a highest possible mini-batch size of 64 limited by GPU memory constraints was chosen so that bad training data examples wouldn't affect the training process.

Taking advantage of the advection fields' equivariance to rotations, the data was augumented by applying random rotations to the mini-batches.
With this, overfitting to the training data was avoided while permitting a smaller training dataset and shorter training times.

\subsubsection{Training Results}
The network was trained on the Finnish IT Center for Science (CSC) Puhti high performance computing environment using a single Nvidia V100 GPU with \SI{32}{\giga\byte} of VRAM.
The network was implemented using the Python library PyTorch wrapped in the PyTorch Lightning interface.

The network was trained with the training data until the average MSE loss metric for the validation set reached a plateau.
Remarkably, the training is completed after only about 10 minutes and after around 10 epochs.
The efficiency of the training process can be attributed to the simplicity of the network architecture and the relatively small dataset required for good performance.

To validate the model's capability to generalise, it was applied to simple synthetic data patterns not similar to the training data.
Despite the immense differences to the actual cloud images, the model is able to successfully capture correct advection patterns given similar scaling of the data to the training dataset.
Figure~\ref{fig:synthetic_example} shows model predictions for a Gaussian circle and squares.

To place the mode's performance in a broader context, it was compared against two prediction methods with the test part of the dataset: the well established Lucas-Kanade technique for optical flow estimation implemented in the PySteps library~\cite{lucaskanade}, and a naive prediction using the most recent observation as the forecast.
As predictability for the data especially over multiple time steps is known to be poor, the naive prediction serves as a rudimentary way to assess whether the advective models are able to capture any meaningful dynamics in the data.
The Lucas-Kanade method provides a good baseline for an advective model as it is a commonly used computational method in similar meteorological applications.
It assumes a locally constant advection field that is solved using least squares approximation at each pixel using local spatial and temporal gradients.

In addition to visual estimations of results, qualitative evaluations were done based three metrics: mean absolute error (MAE), root mean squared error (RMSE), and fractions skill score (FSS).
MAE computes the mean of the absolute deviations of the predicted value $\hat{y}$ from the target $y$:
\begin{equation*}
    MAE = \frac{1}{N} \sum_{i=1}^{N} \vert \hat{y}_i - y_i \vert,
\end{equation*}
and the RMSE computes the square root of the square deviations and thus gives more emphasis on larger errors: 
\begin{equation*}
    RMSE = \sqrt{\frac{1}{N} \sum_{i=1}^{N} \vert \hat{y}_i - y_i \vert^2}.
\end{equation*}
Hence these two metrics give insight to the average predictive performance across the entire image domain.
Due to the averaging the one-dimensional indexing can be thought here as flattening multidimensional data.
But as the neural network was trained to minimise MSE loss and the Lucas-Kanade method computationally minimises least squares, it is also important to use other metrics to get a less biased perspective on the performance.

FSS is a spatial verification metric introduced in~\cite{fss} that compares predictions and targets on a spatial area of a given size above a given threshold.
FSS first computes binary fields $I_{\hat{y}}$ and $I_{y}$ for the two-dimensional predicted field $\hat{y}$ the target $y$ given the threshold $q$ as
\begin{equation}\label{eq:binaryfields}
    \bm{1}_{\geq q}(\hat{y}) = 
        \begin{dcases}
            1, &\hat{y}_{i,j} \geq q\\
            0, &\hat{y}_{i,j} < q
        \end{dcases}, \quad
     \bm{1}_{\geq q}(y) = 
        \begin{dcases}
            1, &y_{i,j} \geq q\\
            0, &y_{i,j} < q
        \end{dcases}.
\end{equation}
From the binary fields~\eqref{eq:binaryfields} spatial densities for a square window size of $n \times n$ are computed as
\begin{subequations}\label{eq:fssdensities}
    \begin{align}
        I_{\hat{y}}(n,q)_{i,j} &= \frac{1}{n^2} \sum_{k=1}^{n}\sum_{l=1}^{n} \bm{1}_{\geq q}(\hat{y}) \left[i+k-1- \frac{n-1}{2}, j+l-1-\frac{n-1}{2} \right],\\
        I_{y}(n,q)_{i,j} &= \frac{1}{n^2} \sum_{k=1}^{n}\sum_{l=1}^{n} \bm{1}_{\geq q}(y) \left[i+k-1- \frac{n-1}{2}, j+l-1-\frac{n-1}{2} \right].
    \end{align}
\end{subequations}
In principle, the window doesn't need to be a square but as discussed in~\cite{fss}, more sophisticated kernels would create additional complexity while not altering enough the results.
Finally the FSS value given window length of $n$ and threshold $q$ is then computed from the spatial averages~\eqref{eq:fssdensities} by
\begin{equation*}
    FSS(n,q) = 1 - \frac{MSE\left(I_{\hat{y}}(n,q), I_{y}(n,q)\right)}{MSE_{\text{ref}}\left(I_{\hat{y}}(n,q), I_{y}(n,q)\right)},
\end{equation*}
where 
\begin{equation*}
    MSE\left(I_{\hat{y}}(n,q), I_{y}(n,q)\right) = \sum_{i=1}^{n_x} \sum_{i=1}^{n_y} \left(I_{\hat{y}}(n,q)_{i,j} -  I_{y}(n,q)_{i,j}\right)^2,
\end{equation*}
and the reference MSE is the maximum difference
\begin{equation*}
    MSE_{\text{ref}}\left(I_{\hat{y}}(n,q), I_{y}(n,q)\right) = \sum_{i=1}^{n_x} \sum_{i=1}^{n_y}I_{\hat{y}}(n,q)_{i,j}^2 + \sum_{i=1}^{n_x} \sum_{i=1}^{n_y}I_{y}(n,q)_{i,j}^2.
\end{equation*}
Thus the FSS value for a perfect skill where the spatial densities for both of the binary fields agree is 1 and for no skill when the spatial densities disagree completely the value is 0.
Due to the threshold and window size variables, computing multiple FSS scores is benefitial to get a better understanding of the predictive performance of a model.

In order to ensure a meaningful and balanced comparisons, some considerations need to be taken into account when computing the loss metrics.
Firstly, due to the wide-ranging and abstract nature of the raw COT values, the metrics were calculated on the normalised test dataset.
This was essential to provide a more consistent interpretation between both strong and weak cloud patterns given the advection-based modelling focusing on the motion of cloud structures rather than evolution or formation.
Furthermore, acknowledging the inherent limitations of purely advective models, the evaluations were restricted to regions of the predicted images where both the Lucas-Kanade and the neural network generated forecasts based on the latest available state, rather than extrapolating values from outside the image area.
These strategies should ensure a more equitable comparison of the models' advective performance.

Figures~\ref{fig:mae_loss},~\ref{fig:rmse_loss}, and~\ref{fig:fss} visualise the computed averaged metrics of forecasts of the test dataset up to two hours.
Across all the metrics the neural network demonstrates performance comparable to the Lucas-Kanade method for the immediate next time step.
Given that the Lucas-Kanade method has been refined over time and is considered a robust technique in this domain, the neural network model's close performance is an encouraging result and affirms the validity of these kinds of neural network applications.

The naive predictions, while not matching the performance of the advective models, is not significantly lagging.
This relatively close performance can largely be attributed to the challenging nature of the noisy COT dataset marked by difficult predictability and the high share of missing values interpolated in a rudimentary manner.
The latter also underscores a need for more sophisticated, yet efficient methods for handling of missing observations for which the Bayesian filtering approach is one method.
Moreover, as already discussed, a significant portion of the dataset exhibits no advective behaviour thus not providing purposeful data for advective modelling methods.

Despite the challenging dataset, what stands out is the resilience of the neural network model.
This is partly demonstrated by the close performance to the Lucas-Kanade method but also by the ability to generate sensible advection fields with simplified, synthetic examples far from the actual training dataset.
